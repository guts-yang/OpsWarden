## 一、账号与权限管理（共 18 条）

---

**Q001**
**问题：** 账号被冻结了怎么处理？
**解决方案：**
1. 自助解冻：登录运维门户 → 账号中心 → 自助解冻，完成身份验证后自动解冻
2. 人工处理：拨打运维热线 400-XXX-XXXX，工作时间 09:00–18:00
3. 常见冻结原因：连续 5 次密码错误、账号长期未使用（超过 90 天）
4. 解冻成功后建议立即修改密码

---

**Q002**
**问题：** 忘记密码怎么重置？
**解决方案：**
1. 访问运维门户登录页，点击「忘记密码」
2. 输入工号及绑定手机号，接收短信验证码
3. 验证成功后设置新密码（需包含大小写字母 + 数字，至少 8 位）
4. 新密码不能与最近 3 次历史密码相同
5. 若手机号已变更，请联系运维人员人工重置

---

**Q003**
**问题：** 如何申请新系统的访问权限？
**解决方案：**
1. 登录运维门户 → 权限申请模块
2. 选择目标系统及所需权限级别（只读 / 读写 / 管理员）
3. 填写申请原因，选择直属上级作为审批人
4. 审批通过后 24 小时内生效
5. 若 48 小时内未收到通知，请联系运维人员查询审批状态

---

**Q004**
**问题：** 如何为新员工开通运维账号？
**解决方案：**
1. 管理员登录运维门户后台 → 账号管理 → 新建账号
2. 填写工号、姓名、部门、邮箱、手机号
3. 选择权限级别（默认标准用户）
4. 系统自动生成初始密码并通过邮件发送给新员工
5. 新员工首次登录时系统强制要求修改密码

---

**Q005**
**问题：** 员工离职后账号如何处理？
**解决方案：**
1. HR 发起离职流程后，系统自动触发账号冻结
2. 运维人员需在离职当日完成账号冻结确认
3. 冻结后账号数据保留 180 天，期间可申请数据迁移
4. 180 天后账号永久注销，数据归档至审计系统
5. 如需提前注销，需经部门负责人审批

---

**Q006**
**问题：** 如何修改绑定的手机号？
**解决方案：**
1. 登录运维门户 → 个人设置 → 安全信息
2. 点击「修改手机号」，验证旧手机号收到的验证码
3. 输入新手机号并验证新号码收到的验证码
4. 若旧手机号无法接收验证码，需携带工牌到运维部门人工核验后修改

---

**Q007**
**问题：** 账号密码多久需要强制更换一次？
**解决方案：**
1. 系统默认密码有效期为 90 天
2. 到期前 7 天系统会通过邮件和门户消息提醒
3. 到期后强制要求修改，未修改则账号自动锁定
4. 管理员账号密码有效期为 60 天，要求更严格
5. 如需调整密码策略，请联系运维管理员

---

**Q008**

2. 可查看所有已授权系统及对应权限级别

**Q009**
**问题：** 申请的权限审批超时没人处理怎么办？
**解决方案：**
1. 权限申请超过 48 小时未审批，系统自动发送催办通知给审批人
2. 超过 72 小时未审批，自动上报给部门负责人
3. 紧急情况可联系运维热线说明情况，由运维人员协助推进
4. 若审批人出差或请假，可申请更换审批人

**Q012**

1. 系统默认不允许同一账号同时在多个设备登录
2. 如确认是本人操作，点击「强制下线其他设备」继续登录
3. 如非本人操作，立即点击「修改密码」并联系运维人员排查异常登录
4. 运维人员可查看账号登录日志，确认异常来源

5. 管理员账号强制要求开启 2FA

**Q014**

2. 选择「使用备用码」（开启 2FA 时系统会生成 10 个备用码，需提前保存）
3. 若备用码也丢失，联系运维人员进行身份核验后重置 2FA
4. 重置后需重新绑定新的认证设备

## 二、网络与连接问题（共 14 条）

---

**Q019**
**问题：** VPN 连接失败怎么办？
**解决方案：**
1. 检查本机网络是否正常（可先访问公网网站验证）
2. 确认 VPN 客户端版本为最新版（下载地址：运维门户 → 软件下载）
3. 尝试切换 VPN 服务器节点（北京 / 上海 / 广州）
4. 关闭客户端，完全退出后重新启动
5. 若以上无效，使用 `ping vpn.company.com` 测试连通性，截图报障

---

**Q020**
**问题：** 连接 VPN 后无法访问内网资源？
**解决方案：**
1. 确认 VPN 状态显示「已连接」而非「连接中」
2. 检查目标内网 IP/域名是否在 VPN 路由表中（运维门户可查）
3. 尝试 `ping 192.168.1.1`（内网网关）验证内网连通性
4. 部分资源需要额外申请内网访问权限，检查权限是否已授予
5. 重启 VPN 连接，若仍无效提交工单并提供目标地址和报错截图

---

**Q021**
**问题：** 办公网络访问速度很慢怎么排查？
**解决方案：**
1. 使用 `speedtest` 或访问测速网站测试当前带宽
2. 检查是否有大文件传输或备份任务占用带宽
3. 尝试更换网线或重启无线路由器
4. 使用 `tracert` 命令追踪到目标地址的网络路径，找到延迟高的节点
5. 若问题持续，联系网络运维提供 tracert 结果

---

**Q022**
**问题：** 内网 DNS 解析失败，域名无法访问？
**解决方案：**
1. 检查本机 DNS 配置是否为内网 DNS 服务器地址（192.168.X.X）
2. 尝试 `nslookup 目标域名 DNS服务器IP` 验证 DNS 解析
3. 清除本地 DNS 缓存：Windows 执行 `ipconfig /flushdns`，Mac 执行 `sudo dscacheutil -flushcache`
4. 若 DNS 服务器无响应，联系网络运维检查 DNS 服务状态
5. 临时解决：在 hosts 文件中手动添加域名解析记录

---

**Q023**
**问题：** 无线网络频繁断开连接怎么处理？
**解决方案：**
1. 检查无线网卡驱动是否为最新版本
2. 修改无线网络设置：电源管理 → 关闭「允许计算机关闭此设备以节省电源」
3. 尝试固定连接到 5GHz 频段（信号更稳定）
4. 检查 AP 信号强度，信号弱于 -70dBm 时建议移动位置或申请增加 AP
5. 若多人同时断网，联系网络运维检查 AC 控制器状态

---

**Q024**
**问题：** 远程桌面（RDP）连接失败？
**解决方案：**
1. 确认目标机器已开启远程桌面功能：系统属性 → 远程 → 允许远程连接
2. 检查防火墙是否放通 3389 端口
3. 确认使用的账号有远程登录权限
4. 若通过 VPN 访问，确认 VPN 已正常连接
5. 测试命令：`telnet 目标IP 3389`，若超时说明端口未放通

---

**Q025**
**问题：** SSH 连接服务器超时或被拒绝？
**解决方案：**
1. 确认服务器 SSH 服务正在运行：`systemctl status sshd`
2. 检查本机 IP 是否在服务器 SSH 白名单内
3. 测试端口连通性：`telnet 服务器IP 22`
4. 使用 `ssh -v 用户名@服务器IP` 查看详细连接日志
5. 若提示「Permission denied」，检查密钥或密码是否正确
6. 联系运维人员确认防火墙规则是否放通你的 IP

---

**Q026**
**问题：** 防火墙申请放通端口的流程是什么？
**解决方案：**
1. 提交工单：说明源 IP/网段、目标 IP/网段、端口号、协议（TCP/UDP）、业务用途
2. 经部门负责人审批
3. 网络运维人员审核后配置防火墙规则
4. 配置完成后通知申请人验证
5. 临时规则有效期默认 30 天，长期需求请在申请时注明「永久」

---

**Q027**
**问题：** 如何申请固定 IP 地址？
**解决方案：**
1. 提交工单：提供设备 MAC 地址、设备位置、用途说明
2. 网络运维在 DHCP 服务器为该 MAC 地址绑定固定 IP
3. 固定 IP 从运维管理的 IP 段中分配，不可自行修改
4. 服务器类设备建议通过此方式申请固定 IP，避免 IP 冲突

---

**Q028**
**问题：** 两台设备出现 IP 地址冲突怎么处理？
**解决方案：**
1. 在受影响设备上执行 `arp -a` 找到冲突 IP 对应的 MAC 地址
2. 联系网络运维，提供冲突 IP 和两台设备的 MAC 地址
3. 运维人员排查 DHCP 分配记录，找到冲突源头
4. 临时处理：将其中一台设备的 IP 手动改为其他地址
5. 根本解决：网络运维修正 DHCP 分配策略

---

**Q029**
**问题：** 如何测试两台服务器之间的网络带宽？
**解决方案：**
1. 确认两台服务器均已安装 `iperf3`（未安装：`yum install iperf3` 或 `apt install iperf3`）
2. 服务端执行：`iperf3 -s`
3. 客户端执行：`iperf3 -c 服务端IP -t 30`（测试 30 秒）
4. 结果中 `Bitrate` 即为实测带宽
5. 测试完成后关闭服务端 iperf3 进程

---

**Q030**
**问题：** 公司网络能上网但特定网站无法访问？
**解决方案：**
1. 检查该网站是否在公司网络访问策略的黑名单中
2. 确认是否为 DNS 解析问题：`nslookup 目标网站` 查看解析结果
3. 业务需要访问的网站可提交工单申请加入白名单，注明业务用途
4. 个人娱乐类网站受访问策略限制，不予开放
5. 紧急情况可使用手机热点临时解决

---

**Q031**
**问题：** 网络打印机无法连接？
**解决方案：**
1. 确认打印机已开机且网络指示灯正常
2. 在电脑上 ping 打印机 IP（打印机 IP 在设备标签或配置页上查看）
3. 若 ping 不通，检查打印机是否连接到正确的网络交换机端口
4. 重新添加打印机：控制面板 → 设备和打印机 → 添加打印机 → 输入打印机 IP
5. 若仍无法连接，联系运维人员检查打印机网络配置

---

**Q032**
**问题：** 如何配置代理服务器访问外网？
**解决方案：**
1. 代理服务器地址请从运维门户 → 网络配置页面获取
2. Windows：设置 → 网络和Internet → 代理 → 手动代理设置，填入地址和端口
3. Mac：系统偏好设置 → 网络 → 高级 → 代理，勾选并填写 HTTP 代理
4. 浏览器代理（Chrome）：设置 → 系统 → 打开您计算机的代理设置
5. 不同业务系统的代理需求不同，如有疑问联系网络运维

## 三、服务器与系统运维（共 16 条）

---

**Q033**
**问题：** 服务器 CPU 使用率持续 100% 怎么处理？
**解决方案：**
1. 执行 `top` 或 `htop` 找出 CPU 占用最高的进程
2. 记录进程名和 PID，分析是否为正常业务进程
3. 若为异常进程：先记录信息再谨慎处理，执行 `kill -9 PID`（需确认安全）
4. 若为正常业务进程负载过高：联系应用负责人评估是否需要扩容
5. 检查是否有定时任务在同一时间集中执行，错开执行时间

---

**Q034**
**问题：** 服务器内存不足，系统出现 OOM（内存溢出）怎么处理？
**解决方案：**
1. 执行 `free -h` 查看当前内存使用情况
2. 执行 `ps aux --sort=-%mem | head -20` 找出内存占用最高的进程
3. 检查 `/var/log/messages` 或 `dmesg` 查看 OOM Killer 日志，确认被杀死的进程
4. 临时处理：重启内存占用异常的服务
5. 长期解决：分析内存泄漏原因或申请扩容服务器内存

---

**Q035**
**问题：** 服务器磁盘空间不足怎么处理？
**解决方案：**
1. 执行 `df -h` 查看各分区使用情况
2. 执行 `du -sh /* | sort -rh | head -20` 找出占用空间最大的目录
3. 清理方向：日志文件（`/var/log`）、临时文件（`/tmp`）、旧备份文件
4. 压缩或归档大文件：`tar -czf archive.tar.gz 目录`
5. 若清理后仍不足，提交工单申请磁盘扩容

---

**Q036**
**问题：** 如何查看服务器的系统日志？
**解决方案：**
1. 系统日志：`/var/log/messages`（CentOS）或 `/var/log/syslog`（Ubuntu）
2. 使用 `tail -f /var/log/messages` 实时查看最新日志
3. 使用 `grep "ERROR" /var/log/messages` 过滤错误日志
4. 查看最近启动日志：`journalctl -b`
5. 查看特定时间段日志：`journalctl --since "2026-03-01" --until "2026-03-02"`

---

**Q037**
**问题：** Linux 服务器时间不准确怎么同步？
**解决方案：**
1. 检查当前时间：`date`
2. 手动同步 NTP：`ntpdate ntp.company.com` 或 `chronyc makestep`
3. 检查 NTP 服务状态：`systemctl status chronyd`（或 `ntpd`）
4. 若 NTP 服务未运行：`systemctl start chronyd && systemctl enable chronyd`
5. 验证时间同步状态：`chronyc tracking`

5. 修改后重载：`systemctl reload sshd`，并确认新配置下可以正常登录再关闭旧连接

**Q073**
**问题：** 如何处理 SSL/TLS 证书被标记为不信任的问题？
**解决方案：**
1. 检查证书链是否完整（缺少中间证书是常见原因）
2. 查看证书详情：`openssl s_client -connect 域名:443 -showcerts`
3. 下载并配置完整证书链（通常为 domain.crt + intermediate.crt 合并文件）
4. 检查证书是否过期：`openssl x509 -in cert.pem -noout -dates`
5. 若为自签名证书导致不信任，需在客户端导入 CA 根证书，或替换为 CA 签发的正式证书

---

**Q074**
**问题：** 如何合规地销毁服务器上的敏感数据？
**解决方案：**
1. 普通文件删除（`rm`）不安全，数据可被恢复
2. 使用 `shred` 命令安全覆写：`shred -vzn 3 文件名`（覆写 3 次后删除）
3. 整个磁盘安全清除：`dd if=/dev/urandom of=/dev/sda bs=1M`（耗时较长）
4. SSD 磁盘：使用 ATA Secure Erase 命令，通过 `hdparm` 工具执行
5. 物理销毁（退役设备）：联系专业数据销毁机构，获取销毁证明

## 六、监控与告警（共 10 条）

---

**Q075**
**问题：** 收到 CPU 高负载告警怎么处理？
**解决方案：**
1. 登录告警服务器，执行 `top` 确认 CPU 使用情况
2. 找出占用 CPU 最高的进程，判断是否为正常业务
3. 若为异常进程：记录并上报，谨慎处理
4. 若为正常业务突发：联系应用负责人评估是否需要扩容或限流
5. 在监控平台查看历史趋势，判断是突发还是持续增长
6. 处理完成后在告警工单中记录处理结果

---

**Q076**
**问题：** 磁盘使用率超过 80% 的告警如何处理？
**解决方案：**
1. 登录服务器执行 `df -h` 确认具体分区使用情况
2. 执行 `du -sh /var/log/* | sort -rh | head -10` 找出大文件
3. 清理策略：
   - 清理过期日志：`find /var/log -name "*.gz" -mtime +30 -delete`
   - 清理临时文件：`rm -rf /tmp/*`
   - 清理已归档的备份
4. 若清理后仍超过 85%，提交磁盘扩容申请
5. 设置告警阈值梯度：80% 警告，90% 紧急，95% 自动告警升级

---

**Q077**
**问题：** 服务健康检查失败告警如何响应？
**解决方案：**
1. 登录监控平台查看具体失败的健康检查节点
2. 尝试手动访问健康检查接口：`curl http://服务IP:端口/health`
3. 查看服务日志确认故障原因
4. 常见原因：服务崩溃（重启服务）、依赖组件故障（排查依赖）、资源不足（清理或扩容）
5. 修复后确认健康检查连续通过 3 次后关闭告警

---

**Q078**
**问题：** 如何配置新的监控告警规则？
**解决方案：**
1. 登录监控平台（Prometheus/Zabbix/云监控）
2. 进入告警规则配置页面，选择目标指标
3. 配置告警条件：指标名、阈值、持续时间（避免抖动误报）
   示例：CPU 使用率 > 80% 持续 5 分钟才触发告警
4. 配置告警通知：邮件组 / 飞书群 / 短信
5. 配置告警分级（P1 紧急 / P2 高 / P3 中 / P4 低）

---

**Q079**
**问题：** 告警太多形成「告警疲劳」怎么治理？
**解决方案：**
1. 审查现有告警规则，删除长期误报或无人处理的告警
2. 调整告警阈值和持续时间，减少抖动告警
3. 合并相关告警：同一服务的多个指标告警聚合为一条
4. 引入告警分级，P3/P4 告警仅记录不通知，P1/P2 才触发人工
5. 每月进行告警复盘，持续优化告警质量

---

**Q080**
**问题：** 如何监控某个接口的响应时间？
**解决方案：**
1. 简单测试：`curl -o /dev/null -s -w "%{time_total}" http://接口地址`
2. 在监控平台配置 HTTP 探针，定期请求接口并记录响应时间
3. 设置响应时间告警阈值（建议 P95 响应时间 > 2s 时告警）
4. 使用 APM（如 Skywalking/Jaeger）进行链路追踪，定位慢接口原因
5. 定期分析接口性能趋势报告，提前发现性能劣化

---

**Q081**
**问题：** 日志监控平台没有收到日志怎么排查？
**解决方案：**
1. 检查日志采集 Agent（Filebeat/Fluentd）是否正常运行
2. 查看 Agent 日志确认是否有报错：`journalctl -u filebeat -n 50`
3. 检查日志文件路径配置是否正确，文件是否有读权限
4. 确认网络是否可以连通到日志收集服务器（Elasticsearch/Kafka）
5. 检查日志收集服务端状态，确认存储空间未满

---

**Q082**
**问题：** 如何设置业务指标的监控大盘？
**解决方案：**
1. 在 Grafana 中新建 Dashboard
2. 确定核心业务指标：QPS、错误率、响应时间、在线用户数等
3. 为每个指标创建对应 Panel，配置 PromQL 或数据源查询
4. 布局建议：黄金指标（4个）置顶 → 资源指标 → 业务明细
5. 配置 Dashboard 自动刷新（30s~1min）并设为首页

---

**Q083**
**问题：** 如何处理夜间值班期间收到的 P1 告警？
**解决方案：**
1. 立即响应：P1 告警要求 5 分钟内响应，15 分钟内给出初步判断
2. 确认告警有效性：登录系统查看实际影响，排除误报
3. 初步处置：重启服务、切换备用节点等快速止血措施
4. 拉起应急群：通知相关负责人，包括应用负责人、DBA（如需）
5. 处置完成后记录值班日报，包括：告警原因、处置过程、后续改进项

---

**Q084**
**问题：** 服务器网卡流量异常告警如何处理？
**解决方案：**
1. 登录服务器查看实时网络流量：`iftop` 或 `nload`
2. 查看连接详情，找到流量来源：`ss -antp | sort -k5 -rn | head -20`
3. 若为正常业务峰值：评估是否需要扩容带宽或优化流量
4. 若为异常外联（数据上传到陌生 IP）：立即隔离并上报安全团队
5. 在防火墙临时限制异常 IP 的访问

## 七、办公设备与环境（共 10 条）

---

**Q085**
**问题：** 电脑开机后显示蓝屏（BSOD）怎么处理？
**解决方案：**
1. 记录蓝屏时显示的错误代码（如 `IRQL_NOT_LESS_OR_EQUAL`）
2. 尝试重启，若能正常进入系统：运行 `sfc /scannow` 检查系统文件
3. 若重复出现：
   - 检查最近安装的驱动或软件，尝试卸载
   - 进入安全模式排查
4. 若无法进入系统：使用 Windows PE 启动盘检查磁盘
5. 以上无效则提交工单，运维人员上门处理，准备好备用机器

---

**Q086**
**问题：** 电脑运行越来越慢怎么处理？
**解决方案：**
1. 检查开机自启程序：任务管理器 → 启动选项卡，禁用不必要的启动项
2. 清理磁盘：右键 C 盘 → 属性 → 磁盘清理，清理临时文件
3. 检查内存和 CPU 占用：任务管理器查看占用高的进程
4. 检查是否有恶意软件：运行企业杀毒软件全盘扫描
5. 若问题持续，联系运维人员评估是否需要重装系统或更换硬件

---

**Q087**
**问题：** 显示器无信号或画面异常怎么处理？
**解决方案：**
1. 检查视频线连接是否松动，重新插拔
2. 尝试切换显示器的输入源（HDMI/DP/VGA）
3. 更换视频线测试是否为线材故障
4. 将显示器连接到其他电脑，确认是显示器问题还是电脑问题
5. 若为显示器故障，联系运维人员申请更换；若为显卡问题，报修处理

---

**Q088**
**问题：** 公司统一发放的软件如何安装？

1. 打开企业软件分发平台（运维门户 → 软件中心）
2. 搜索所需软件，点击「安装」
3. 软件会自动推送到你的电脑并静默安装（约5-15分钟）
4. 若需特殊软件（软件中心没有），提交工单说明业务用途，由运维评估后统一推送
5. 禁止私自安装未经审批的软件，存在安全风险

**Q089**

**解决方案：**

2. 填写申请理由和设备规格需求
3. 申请需经部门负责人审批
4. 审批通过后运维人员在 3 个工作日内配发
5. 故障设备需交回运维部门登记

---

**Q090**
**问题：** 会议室的投影仪或大屏无法连接电脑？
**解决方案：**
1. 检查视频线连接（HDMI/Type-C），确认接口类型匹配
2. Windows：按 `Win+P` 选择显示模式（复制/扩展）
3. Mac：系统偏好设置 → 显示器，检查检测到的显示器
4. 若无法识别：更换视频线或转接头测试
5. 仍无效：使用会议室的无线投屏功能（Miracast/AirPlay），或联系运维人员现场支持

---

**Q091**
**问题：** 电脑无法识别 USB 设备怎么处理？
**解决方案：**
1. 更换 USB 接口（有时特定接口有问题）
2. 在设备管理器中刷新/重新扫描硬件
3. 卸载并重新安装 USB 控制器驱动
4. 检查企业安全策略是否限制了 USB 存储设备（部分企业禁用 U 盘）
5. 若为授权设备（如安全 U 盘），联系运维人员添加设备白名单

**Q092**
**问题：** 如何远程协助同事解决电脑问题？
**解决方案：**
1. 使用企业统一部署的远程工具（运维门户 → 远程协助）
2. 被协助方打开工具，点击「获取连接码」并告知协助人
3. 协助人输入连接码后申请控制权限
4. 被协助方同意后可远程操作
5. 所有远程协助操作均有日志记录，请注意操作规范

---

**Q093**
**问题：** 打印机提示「脱机」无法打印怎么处理？
**解决方案：**
1. 检查打印机电源和网络连接
2. Windows：控制面板 → 设备和打印机 → 右键打印机 → 查看正在打印的内容 → 取消「脱机使用打印机」勾选
3. 清除打印队列中卡住的任务：停止 Print Spooler 服务 → 清除 `C:\Windows\System32\spool\PRINTERS` 中的文件 → 重启服务
4. 重新安装打印机驱动
5. 仍无效则 ping 打印机 IP，确认网络连通性

**Q094**

3. 申请需提前至少 3 个工作日提交，确保运维有时间准备
4. 运维人员完成设备配置（系统安装、软件预装、账号创建）后通知领取
5. 员工领取时签署设备领用单

**Q095**

5. 若系统本身故障，联系运维人员检查应用服务状态

**Q096**

**Q098**
