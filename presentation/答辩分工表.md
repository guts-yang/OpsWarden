## 四、视频录制分工

### 4.1 主演示视频（自学习闭环主线）——丁其彬

> 脚本见 `演示视频脚本.md`，一镜到底：命中 → 未命中建单 → 解决回写 → 再问命中。整段由 **丁其彬** 录制 + 剪辑，控制在 90 秒内。

### 4.2 功能短片（每个功能一个 5–10 秒"动图"，嵌入 PPT 对应页）

> 形式：无声、循环播放的短片（GIF 或静音 mp4，reveal.js 用 `<video autoplay loop muted>`），放在介绍该功能的幻灯片上，做"动图"效果。
> 录制规范：1920×1080、浏览器全屏、鼠标放慢、单一动作干净利落、5–10 秒可循环。

> 已在 `index.html` 第 10 页下方建好 6 个"功能动图"子页（向下翻页 ↓），每页含一个 `<video autoplay loop muted>` 占位，文件就位即自动循环播放。
> 文件统一放 `presentation/assets/clips/`，命名 `c1.mp4 … c10.mp4`（无声、可循环、5–10 秒）。

| 编号 | 功能短片 | 放置幻灯片 | 文件 | 时长 | 录制 |

## 五、答辩问答分工（三人一起回答）

> 答辩时三人同时在线，按提问领域分工接话，**每个人必须提前准备好自己负责的模块**。

| 提问领域 | 主答 | 准备要求 |
|----------|------|----------|
| RAG / 向量检索 / embedding / DeepSeek / Agent | 廖晨扬 | 两阶段检索、量化锚点、自学习闭环、降级策略 |
| 账号 / 工单 / 数据库 / 统一响应 / JWT 鉴权 | 吴雨彤 | 各接口实现原理、表结构、状态流转 |
| Docker / 部署 / 云服务器 / 域名 / 敏捷协作 (Linear) | 丁其彬 | 部署流程、容器编排、Linear 看板与 GitHub 集成 |
| 前端 / 页面交互 / 权限路由 | 廖晨扬（吴雨彤辅助） | 路由守卫、角色权限、Axios 响应拦截 |

**关键准备要求（务必落实）**

- 吴雨彤、丁其彬**必须理解除 RAG 以外的所有接口实现原理**：
  `auth`（登录/注册）、`account`（CRUD/冻结/重置）、`ticket`（自动/手动建单、解决、回写、关闭、日志）、`analytics`（仪表盘统计）、`knowledge`（增删改查、stats）。
- 对照文件梳理：`backend/app/api/` 各路由 + `backend/app/models/` + `init.sql`。
- 如有余力，**建议把 RAG 底层也熟悉一遍**（`backend/app/rag/`：embedder / retriever / quantizer / llm），答辩时可互相补位。
