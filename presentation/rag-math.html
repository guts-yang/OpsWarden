总览：RAG 在数学上求解什么

一个最近邻检索 + 条件生成的复合问题

给定用户问题 \(q\) 与知识库 \(\mathcal{D}=\{d_1,\dots,d_N\}\)，目标是先选出最相关的 \(K\) 条，再据此生成答案：

主目标
\[ \text{answer} \;=\; \text{LLM}\Big(q,\; \underset{d \in \mathcal{D}}{\text{TopK}}\;\text{sim}(q, d)\Big) \]
★
全部精巧之处都在于：如何又快又准地计算相似度 \(\text{sim}\) 并取出 TopK。下面 6 步逐一拆解。
1

文本 → 向量（Embedding）

代码：embedder.py

一段文字 \(t\) 经 BGE 模型 \(f(\cdot)\) 映射成 512 维实向量，并做 L2 归一化：

\[ \mathbf{v} = \frac{f(t)}{\lVert f(t)\rVert_2},\qquad \lVert\mathbf{v}\rVert_2 = \sqrt{\sum_{i=1}^{512} v_i^2} = 1 \]
i
为什么归一化？ 归一化后所有向量落在单位超球面上，此时点积直接等于余弦相似度（见第 3 步），免去每次重算模长的开销。代码即 normalize_embeddings=True。

非对称编码（关键技巧）：查询与文档用不同方式编码——查询加 instruction 前缀，文档不加：

\[ \mathbf{q} = f(\underbrace{\text{“为这个句子生成表示以用于检索相关文章：”}}_{\text{instruction 前缀}} + q),\qquad \mathbf{d}=f(d) \]

前缀让模型知道「这是检索查询」，把查询向量推向更利于召回的语义子空间，实测提升召回率。

2

知识入库：联合向量 + 自检质量分

代码：retriever.py · ingest_kb_entry()

每条知识是「问题 \(Q\) + 解决方案 \(S\)」对。OpsWarden 存的不是单一问题向量，而是联合表示：

\[ \mathbf{q}_Q=f(Q),\quad \mathbf{q}_S=f(S),\qquad \mathbf{e}=\frac{\tfrac12(\mathbf{q}_Q+\mathbf{q}_S)}{\big\lVert \tfrac12(\mathbf{q}_Q+\mathbf{q}_S)\big\rVert_2} \]
▲
几何意义：联合向量落在 \(\mathbf{q}_Q\) 与 \(\mathbf{q}_S\) 的「角平分线」方向。无论用户问法更像「问题」还是更像「答案描述」，都能被检索到，召回更全面。

同时计算自检质量分（写入 match_score）——问题向量与方案向量的余弦相似度，裁剪到 \([0,1]\)：

\[ \text{match\_score} = \operatorname{clip}\!\big(\mathbf{q}_Q\cdot\mathbf{q}_S,\;0,\;1\big) \]

它衡量「问题与答案语义是否自洽」，是数据治理指标，与实际检索命中率无关。

3

相似度 = 余弦 = 点积

代码：retriever.py · search()

两向量 \(\mathbf{a},\mathbf{b}\) 的余弦相似度：

\[ \text{sim}(\mathbf{a},\mathbf{b})=\cos\theta=\frac{\mathbf{a}\cdot\mathbf{b}}{\lVert\mathbf{a}\rVert_2\,\lVert\mathbf{b}\rVert_2}=\frac{\sum_i a_ib_i}{\sqrt{\sum_i a_i^2}\,\sqrt{\sum_i b_i^2}} \]

因第 1 步已归一化（\(\lVert\mathbf a\rVert=\lVert\mathbf b\rVert=1\)），它退化为简单点积：

\[ \text{sim}(\mathbf{a},\mathbf{b})=\mathbf{a}\cdot\mathbf{b}\in[-1,1] \]
i
pgvector 返回的是余弦距离，二者关系：\(\;\text{distance}=1-\cos\theta\;\Rightarrow\;\text{score}=1-\text{distance}\)。距离越小 = 相似度越高。代码即 (1 - distance_expr)。
4

两阶段检索（系统的灵魂）

代码：quantizer.py + retriever.py · search()

对全部 \(N\) 条算精确余弦是 \(O(N)\)，数据一多就慢。所以拆成 粗筛 + 精排。

4.1　向量量化：把相似向量「吸附」成锚点

把连续向量吸附到间距为 \(\varepsilon\) 的网格（\(\varepsilon=0.02\)）：

\[ \mathbf{a}=\operatorname{round}\!\Big(\frac{\mathbf{v}}{\varepsilon}\Big)\cdot\varepsilon,\qquad \text{quant\_key}=\operatorname{SHA256}\big(\text{bytes}(\mathbf{a})\big) \]

几乎相同的向量被吸附到同一锚点，相同 key 共享锚点（幂等 upsert）。相当于给知识库做了一层聚类分桶，索引规模大幅压缩。

4.2　L1 粗筛：锚点路由

只对锚点集合算余弦距离（借助 pgvector 近似索引），取最近 8 个：

\[ \mathcal{A}_{\text{top}}=\underset{\mathbf{a}\in\text{Anchors}}{\arg\text{top-}8}\;\big(1-\text{cosine\_distance}(\mathbf{q},\mathbf{a})\big) \]

对应 RAG_ANCHOR_TOP_K = 8，把搜索范围从「全库」缩到「8 个桶里的少量候选」。

4.3　L2 精排：全精度重排

取出候选条目 \(\mathcal{C}\)，用未量化的原始向量逐条精算余弦：

\[ s_i=\frac{\mathbf{q}\cdot\mathbf{e}_i}{\lVert\mathbf{q}\rVert_2\,\lVert\mathbf{e}_i\rVert_2},\qquad i\in\mathcal{C} \]
★
为什么两段？量化会丢精度（快但不准），所以只用它「缩范围」；最终排序仍用全精度——先粗筛保证快，再精排保证准。
5

阈值判定：命中 / 未命中

代码：search() + chat_pipeline.py

过滤、降序、取前 3：

\[ \mathcal{R}=\underset{k=3}{\text{TopK}}\Big(\{\,i\in\mathcal{C}\;:\;s_i\ge\tau\,\}\Big),\qquad \tau=0.40 \]

判定规则：

\[ \text{结果}=\begin{cases}\textbf{命中}\to\text{交给 LLM 生成}, & \max_i s_i\ge 0.40\\[4pt]\textbf{未命中}\to\text{通用回答 / 建工单}, & \max_i s_i<0.40\end{cases} \]

τ 调高 ↑

更准，但可能漏召回 → 误判未命中、多建工单。

τ 调低 ↓

召回更多，但可能答非所问。

\(\tau=0.40\)（RAG_SCORE_THRESHOLD）是召回率与准确率的权衡旋钮。

6

条件生成（LLM）

代码：llm.py · generate_answer()

把 Top-K 条目拼成上下文 \(C\)，让 DeepSeek 在该上下文条件下生成：

\[ \text{answer}=\arg\max_{y}\;\prod_{t}P_\theta\big(y_t\mid y_{

采样温度 \(T=0.1\)（很低），数学上让 softmax 分布更「尖锐」：

\[ P(y_t=w)=\frac{\exp(z_w/T)}{\sum_{w'}\exp(z_{w'}/T)} \]
i
\(T\) 越小，分布越集中在最高概率 token → 输出越确定、越少胡编，符合运维场景对稳定性的要求。兜底：无 API Key 时直接返回 results[0]["solution"] 原文，系统绝不崩。
↺

一句话串起来 + 自学习闭环

从提问到「越用越聪明」的数学本质
端到端求解链
\[ \underbrace{q}_{\text{问题}}\xrightarrow{\text{BGE+前缀}}\mathbf{q}\xrightarrow{\text{量化锚点 Top-8}}\mathcal{C}\xrightarrow{\text{全精度余弦 Top-3}}\mathcal{R}\xrightarrow{\,s\ge0.4\,?\,}\begin{cases}\text{是}\Rightarrow\text{DeepSeek 生成}\\\text{否}\Rightarrow\text{建工单}\end{cases} \]

自学习闭环：把「未命中 → 已解决的工单」再走一遍第 1、2、4.1 步（ingest_kb_entry：编码 → 联合向量 → 量化建锚点）写回知识库。于是下次同样的 \(q\) 在第 5 步就满足 \(s\ge0.4\) 而命中。

用户提问→ RAG 未命中→ 确认后建工单→ 运维解决→ 回写知识库↺ 下次直接命中
★
「越用越聪明」的数学本质：不断往 \(\mathcal{D}\) 里增加能提高 \(\max_i s_i\) 的新点，让原本低于阈值的问题被新增知识点拉过 \(\tau\)。
