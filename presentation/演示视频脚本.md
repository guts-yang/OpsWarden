# OpsWarden 操作演示视频脚本（分镜 + 旁白）

> 用途：答辩第 11 页插播 / 现场演示备份。**总时长目标 90 秒（最长不超过 2 分钟）**。
> 录制建议：1920×1080，关闭无关通知，浏览器全屏，鼠标轨迹放慢，关键操作处停顿 1 秒。
> 一条主线讲透**自学习闭环**：命中 → 未命中建单 → 解决回写 → 再问命中。

---

## 录制前准备（Checklist）

- [ ] 后端已启动：`cd backend && python -m uvicorn app.main:app --host 0.0.0.0 --port 8000`
- [ ] 前端已启动：`cd frontend && npm run dev`（http://localhost:5173）
- [ ] 数据库已初始化，FAQ 知识库已自动导入（首次启动 BGE 模型会下载，提前跑一次）
- [ ] Ollama 已启动且 `OLLAMA_MODEL` 与 `ollama list` 一致（保证生成式回答效果）
- [ ] 准备两个账号：`admin` + 你在 `init.sql` 中设置的密码（演示后台），可另备一个 `user` 账号演示纯问答视角
- [ ] 想好一个"知识库里有的问题"和一个"冷门问题"，提前验证过命中/未命中表现
- [ ] 浏览器缩放 100%，清空 AI 问答历史

## 分镜脚本

### 镜头 0 · 片头（0:00–0:05，5s）
- **画面**：登录页 `LoginView`，输入 admin / 你的管理员密码登录。
- **旁白**：「这是 OpsWarden 运维数字员工，我们登录进系统。」

### 镜头 1 · 场景①命中问答（0:05–0:25，20s）★核心卖点
- **画面**：进入 `AI 问答` 页面 → 点一个快捷提问，或输入一个**知识库里有的问题**（如"如何申请系统访问权限"）→ 回车。
- **要点特写**：等待 1 秒，AI 返回答案；**镜头给到"知识库命中"绿色徽标**。
- **旁白**：「我问一个常见运维问题。注意看——秒级返回标准答案，右上角'知识库命中'。这背后是我们的**两阶段向量检索**，先量化锚点粗筛，再全精度精排。」

### 镜头 2 · 场景②未命中建单（0:25–0:50，25s）★核心卖点
- **画面**：输入一个**冷门、知识库没有的问题**（如"测试环境 Redis 集群脑裂怎么处理"）→ 回车。
- **要点特写**：AI 给出通用建议 + 询问"是否需要创建工单？"→ **点击"确认创建工单"按钮** → 出现"工单已创建 T-2026xxxx-00x"徽标。
- **旁白**：「再问一个知识库里没有的问题。系统识别出未命中，给出通用建议，并征求我同意——**绝不静默建单**。我点确认，工单自动生成，编号在这里。」

### 镜头 3 · 场景③解决并回写知识库（0:50–1:15，25s）★全场高潮
- **画面**：切到 `工单中心 TicketsView` → 打开刚建的工单 → 点"解决" → 填入解决方案 → **勾选"写入知识库"** → 提交。
- **要点特写**：工单状态变为"已解决"；操作日志出现 `writeback` 记录。
- **旁白**：「我以运维身份处理这张工单，填写解决方案，关键一步——勾选'写入知识库'。系统会自动把这条新知识向量化、建立锚点。」

### 镜头 4 · 场景③验证自学习（1:15–1:30，15s）★闭环收尾
- **画面**：回到 `AI 问答`，清空或新开会话 → **再次输入镜头 2 那个冷门问题** → 回车。
- **要点特写**：这次直接命中，返回刚才写入的解决方案，显示"知识库命中"。
- **旁白**：「现在见证奇迹——同样的问题，再问一遍。这次它**直接答出来了**，因为它刚刚学会了。这就是 OpsWarden 的自学习闭环：越用越聪明。」

### 镜头 5 · 收尾（可选，0:00–0:05）
- **画面**：快速扫过 `仪表盘` 数据卡片 / `知识库` 列表里新增的那条。
- **旁白**：「问答、工单、知识库，全程自动闭环。这就是真正能干活的运维数字员工。」

## 旁白风格提示（雷军式）

- 多用「注意看」「见证奇迹」「秒级」「直接就会了」制造期待与爆点。
- 关键数字与卖点处放慢、加重：**两阶段检索**、**绝不静默建单**、**越用越聪明**。
- 每个场景结束给一句"一句话总结"，让评委记得住。
